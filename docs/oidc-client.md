# OIDC Client SDK 実装および検証計画

最終更新: 2026-04-25
対象: `packages/oidc-client-sdk` および検証用環境

---

## 1. 目的
フロントエンドアプリケーション（React, Vue, Next.js 等）が、セキュリティのベストプラクティス（PKCE, Token Rotation 等）を意識せずに IdP と統合できるようにし、その動作を実サーバーで検証する。

## 2. SDK の主な機能
- **認証フロー制御**: `login()`, `logout()`, `handleCallback()` メソッドの提供。
- **PKCE 自動対応**: code_verifier / code_challenge の生成と保存の隠蔽。
- **トークン管理**:
  - メモリ内保存およびサイレントリフレッシュのサポート。
  - セキュアな Cookie ストレージオプションの提供。
- **API インターセプター**: HTTP クライアントにアクセストークンを自動付加するユーティリティ。

## 3. 検証用アプリ (`apps/sample-client`)
SDK の機能を実証するための最小限の Web アプリを構築する。
- **技術**: Vite + TypeScript + SDK
- **検証シナリオ**:
  1. 未ログイン状態でログインボタンを押下 -> IdP ログイン画面へ。
  2. ログイン成功後、アプリに戻りトークンを取得。
  3. 取得したトークンを使って IdP の `/v1/me` API を呼び出し、ユーザー情報を表示。
  4. トークン有効期限切れ後の自動リフレッシュ。

## 4. ステップ別タスク
1. **SDK 開発**: `packages/oidc-client-sdk` のコアロジック実装。
2. **サンプルアプリ構築**: SDK をインポートしたテスト用フロントエンドの作成。
3. **統合テスト**: ローカル Docker 環境で `idp-server` と `sample-client` を並行起動し、E2E での動作確認。
